本 skill 会被 Codex automation 无人值守触发。执行时必须产出下面列出的文件和推送副作用；不要只回复完成。如果任一步骤失败，必须说明具体失败步骤，并停止声称成功。

- 必须写入 `data/last_postmarket_card.md`。

# 定位说明

L4 盘后复盘。**两个核心职责**：

1. **数据落库**：今日 sentiment 写 `sentiment_daily` 表（明日 L1 阶段判定有 10 日基线）+ 同花顺 reason tags 写 `ths_hot_reason` 表（积累题材轮动历史）
2. **复盘 + 明日预案**：对照今日 fact pack 和上午推送的盘前观察池（push_log 表里 msg_id），评估今日主线延续性，给出明日「再战 / 减仓 / 空仓」决策

输出**不是**「今日哪只赚了」，而是「**今日情绪给明日多少能见度 + 哪个主线值得明早盯**」。

# 工作流

每次调用按序执行下面 4 步。

# 输出契约（最重要，违反 = 整体失败）

**所有数据必须有来源支撑**（[[feedback-data-must-be-sourced]]）。

这是**该次卡片唯一允许引用的事实清单**：

- 任何 6 位股票代码 → 必须在 `ALLOWED.codes` keys
- 任何中文股票名（stock_basic 5200 条匹配） → 必须在 `ALLOWED.codes` values
- "N 板/N 连板" → 必须等于 `ALLOWED.lianban[code]`
- "±X.X%" → 必须在 `ALLOWED.pct[code] ± 0.5%`
- "涨停 N 只 / 跌停 N 只 / 炸板 N 只" → 精确等于 `ALLOWED.summary.{limit_up, limit_down, broken}`
- "最高 N 连板" → 等于 `ALLOWED.summary.max_consec`

**禁止**：从训练数据印象/holdings.example.yaml 编出 fact pack 之外的股票（如"大唐发电 6 连板炸板"
当日实际 -0.25%）。**禁止**：把数值"凑整"到容差外（如实际 +2.60% 写成 +3.62%）。

推送脚本 `push.py` 自动跑校验器，违规会被审计日志记录（v1 warn 模式不拦截；v2 enforce 模式直接
拒推）。**写卡前自己对照 ALLOWED 一遍**。

写入文件和 Telegram 推送的内容必须是卡片本身（以 📖 / 🌆 等卡片开头符开头）；Codex automation 最终回复只给简要运行摘要，不要用“完成”替代文件写入和推送。

## Step 1 · 拉今日完整数据 + 写库

```bash
.venv/bin/python .agents/skills/stock-postmarket/scripts/fetch_postmarket.py
```

这一步同时做三件事：

1. 从 akshare + 同花顺拉今日完整收盘数据
2. 计算 phase + 写 `sentiment_daily`（UPSERT，重跑无害）
3. 写 `ths_hot_reason`（含完整 reason tag 个股清单）
4. 渲染 `data/fact_pack/YYYYMMDD_postmarket.md`

涨停池、跌停池、炸板池、强势股池会自动重试；实时源仍失败时，`fetch_postmarket.py` 会回退到 `data/postmarket_cache/` 里当天最近一次成功快照。若四个结构源全部为空且无缓存，脚本会 fail closed，拒绝写入 `limit_up=0` 的假 fact pack。

读完整 stdout。这是今日 fact pack（6 节）：今日情绪快照 / 连板梯队 / 跌停炸板 / 龙虎榜 / 同花顺题材 Top 10 / 近 10 日情绪表。

## Step 1.5 · 风控状态更新（连亏 streak）

执行命令获取连亏状态 JSON：

```bash
.venv/bin/python .agents/skills/stock-postmarket/scripts/update_loss_streak.py
```

返回字段：
- `loss_streak` (int) — 截至今日的连亏天数
- `warn_active` (bool) — 是否触发心态提醒（loss_streak ≥ loss_streak_warn_threshold）
- `recent_pnl` (list) — 最近 5 天浮盈亏明细，用于卡片渲染
- `today_pnl_pct` / `is_loss_today` — 仅日志参考，不进卡片

兜底：脚本失败时返回 `warn_active=false`，本步不阻塞 L4 流程。

## Step 2 · 复盘今日交易决策单

优先复盘 `decision_tickets`，评价系统今天是否真的给出可执行决策。**一条命令出表**：

```bash
.venv/bin/python -m stock_codex.tools.review decision-review
```

输出 markdown 表 + 决策评分（主攻是否触发、潜伏是否触达低吸区、禁买是否规避风险），直接粘进卡片 Step 4 的复盘段。

如果 `decision-review` 报"未找到 decision_tickets"，才回退旧观察池复盘：

```bash
.venv/bin/python -m stock_codex.tools.review review
```

旧观察池判定规则（review.py 内置）：

| 状态 | 判定 | 含义 |
|------|------|------|
| ✅ 触发+收红 | 高 ≥ 买点 且 收 ≥ 买点 | 派别 A 真接力 |
| ⚠️ 触发+假突破 | 高 ≥ 买点 但 收 < 买点 | 上影线，需止损 |
| ❌ 未触发 | 高 < 买点 | 9:50 已放弃 |
| 💥 跌破止损 | 低 ≤ 止损价 | 必须已止损出 |
| ✅/⚪/❌ MA5派 | 按今日涨跌幅分档 | 派别 B/D 无固定买点，看强弱 |

若 review.py 报"未找到今日 premarket 推送"或"解析出 0 只"，回退手动：先 `sqlite3 data/daily.db "SELECT text FROM push_log WHERE source='stock-premarket' ORDER BY id DESC LIMIT 1"` 看推送，再用 akshare `stock_zh_a_spot_em()` 手查代码。回退后必须把失败样本贴给开发者改正则。

> ⚠️ push_log 表的时间列是 `timestamp` ISO 8601 字符串（**不是** `ts` / `created_at`）；若想按今日过滤用 `WHERE date(timestamp)='YYYY-MM-DD'`，而不是 `date(ts,'localtime')` —— 后者直接报错。schema 完整字段见 `stock_codex/schema/init_db.sql` 或 `.schema push_log`。

## Step 2.5 · 当日 + 晚间消息扫描

**继承 [[feedback-news-awareness]]**：题材延续判断不能纯凭涨停/龙虎榜数据，必须叠加消息面。盘后这步覆盖两个窗口：

**窗口 A · 当日完整快讯（09:30 → 现在）**

依次抓（每个源失败就跳）：

1. **WebFetch** `https://www.cls.cn/telegraph` — 财联社电报全日重要快讯（取标星/热度高的 ≥30 条）
2. **WebFetch** `https://www.cls.cn/depth/1003` — 早盘+午盘速递
3. **WebSearch** 限定当日 site:gov.cn / 证监会 / 工信部 / 发改委 政策公告

**窗口 B · 晚间消息 + 隔夜外围预期（15:00 → 现在 + 隔夜预判）**

4. **WebFetch** `https://www.cls.cn/telegraph` — 盘后突发公告（公司临停、重组、业绩预告等）
5. **WebSearch** 主线题材的产业链晚间动态（fact pack Top 3 题材）
6. **WebSearch** 隔夜外围预期：美股期指、中概 ADR、原油黄金、重要海外事件预告

**判定每条消息对今日主线 / 持仓的影响**：
- **延续利好**：命中今日 reason tag Top 3，加权强化"延续"判定
- **退潮利空**：命中今日衰退题材的负面新闻，加权强化"退潮"判定
- **新主线信号**：新政策 / 新产业链事件，今日尚未发酵 → 标记到"新主线"
- **持仓利空**：直接命中 holdings.yaml 任一持仓股的负面新闻 → 必须在卡片明早动作里特别提示

**约束**：每条消息带 URL + 发布时间；24h 窗口；禁用淘股吧/股吧/微博/雪球热议；抓不到明确写"晚间无显著突发"，不要编。

## Step 3 · 综合判定明日主线

读完 Step 1 fact pack 的「五、同花顺题材标签 Top 10」+「四、龙虎榜上榜原因」+「六、近 10 日情绪表」，判断：

### 3a. 题材延续性

对今日 reason tag Top 10 每个题材：

- **延续**：今日该题材 ≥3 只涨停 且 龙虎榜净买入 Top 5 至少有 1 只属于该题材
- **退潮**：今日该题材 ≤1 只涨停，或核心龙头炸板/跌幅 > 5%
- **新主线**：今日 reason 出现次数比昨日翻倍 且 有 ≥3 只联板（连板数 ≥ 2）

### 3b. 情绪阶段切换预判

对照「近 10 日情绪表」最近 3 天：

- 涨停连续 > 60 三天 + 今日炸板率上升 → **明日大概率高位分歧**
- 涨停 < 30 + 跌停 > 30 → **明日延续退潮，建议空仓**
- 涨停从 < 40 跳到 > 60 + 高度梯队补齐 → **明日加速可能**

### 3c. 明日决策（三选一）

| 决策 | 触发条件 | 仓位建议 |
|------|---------|---------|
| **再战** | 阶段稳定在加速 + 主线题材延续 | 总仓位维持，重点跟新主线 |
| **减仓观望** | 高位分歧信号 / 主线退潮 / 触发率 < 50% | 砍 50% 仓位，明早只盯低吸 |
| **空仓** | 退潮 / 跌停 > 30 / 龙头崩盘 | 不接力，等下一波启动信号 |

### 3d. 持仓题材集中度（必做，不可跳过）

读 `holdings.yaml`，对每只持仓提取题材：

- **优先**用 `note` 字段里的题材关键词（如"算电协同二板接力" → 算电/电力）
- 若 note 含混或不清，用今日 fact pack 五（同花顺题材 Top 10）反查该 code 是否归属某主线题材
- 仍不清楚的，标"未分类"

聚合（即使没触发集中也要出一行）：

- 输出分布：`N 只持仓中，题材分布：[电力 ×3, 算力 ×1, 未分类 ×1]`
- 计算 `最大单题材占比 = 最大题材命中数 / 总持仓数`

判定档位：

- `≥ 60%` 同一题材 → ⚠️ 高集中
- `≥ 40%` 且 `< 60%` 同一题材 → 🟡 偏集中
- `< 40%` → ✅ 分散

完全空仓（`holdings.yaml` 为空）：输出一行 `✅ 今日空仓，无集中度风险`，跳过判定。

"明日动作"建议必须和 3a 题材延续性判定结果联动：
- 该主题判为"延续" → "保持"
- 该主题判为"退潮" → "减半至 ≤40%"

最终展示行在 Step 4 卡片 ⚠️ 风险提示段渲染（本节只做分析）。

## Step 4 · 输出 Telegram 卡片

**这张卡片是写给"明天早上要做决策的人"看的，不是数据看板。** 用人话讲今天市场为什么这么走、持仓怎么处理、明天该盯什么。

格式约束（重要）：
- 推送会自动 md→Telegram HTML，**用 `**bold**` 标重点**，不要用 `#` 标题（在 Telegram 会被转粗体但不如直接 emoji+bold 干净）。
- **禁止 markdown 表格**（虽然转换器能转 bullet，但叙事场景下表格会割裂阅读）。用编号列表或缩进。
- 不要堆数字，每个数字要解释"意味着什么"。

模板：

```
🌆 <b>盘后复盘 · [YYYY-MM-DD]</b>

📖 <b>今日市场怎么走</b>

[一段 4-6 句的叙事，必须解释因果链，不是罗列数据]

举例参考写法：
"今天主线还是电力 / 算电协同，大唐发电 5 板封死，但 3 板梯队（康欣 / 京投）盘中开板，
说明承接乏力。龙虎榜算电协同共净买入 X 亿，机构资金还在；但散户接力意愿弱（炸板 23 只，
百邦炸 38 次说明顶部抛压重）。情绪从加速切换到加速末段-分歧前夜。隔夜美股 -0.66% 偏空。"

要点：
- 引用 1-2 个具体数字（涨停数 / 最高连板 / 净买额）但每个都要带解读
- 把今天 vs 昨天的"变化方向"说清楚（资金流向 / 高度梯队 / 题材轮动）
- 一句话定调今日情绪（什么阶段 + 偏多还是偏空）

如果上午 L1 推过观察池且**用户买入了**：

🧠 <b>今日教训</b>（策略沉淀，不立刻改 L1）

1-2 条**今日暴露的具体规律**，写法要求：
- 必须基于今日观察池 / 持仓的真实命中情况，不是套话
- 必须够具体到能做规则（如"3 板封单 < 1 亿次日接力胜率低"），不要"要严格止损"这种泛泛
- **不允许立刻改 L1 SKILL.md** —— 单日样本不足，过拟合昨天比没规则更糟
- 累积 5-10 个交易日后人工汇总，反复出现 ≥3 次的规律才升级进 L1 硬约束

格式：
- <b>规律</b>：[一句陈述]
  <b>证据</b>：[今日哪只票 / 命中表现验证了这条]
  <b>建议升级时机</b>：[再观察 N 个交易日 / 已出现 N 次可升级]

例：
- <b>规律</b>：3 板候选封单 < 1 亿，次日接力胜率显著低于封单 > 2 亿
  <b>证据</b>：今日京投 0.78 亿封单 → 次日跌破止损；大唐 8.6 亿封单 → 继续涨停
  <b>建议升级时机</b>：再观察 5 个交易日，若仍稳定则升级 L1 派别 A 硬约束
