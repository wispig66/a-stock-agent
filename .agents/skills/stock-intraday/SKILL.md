## Codex automation 契约

本 skill 会被 Codex automation 无人值守触发。执行时必须产出下面列出的文件和推送副作用；不要只回复完成。如果任一步骤失败，必须说明具体失败步骤，并停止声称成功。

- 必须根据当前系统时间进入 09:30、09:45、11:30 或 14:30 分支。
- 必须写入 `data/last_intraday_card.md`。
- 必须通过 `.agents/skills/stock-premarket/scripts/push.py --source stock-intraday` 推送。
- 遇到 `PREMARKET_MISSING` 时必须降级报告，不要编造观察池。
- 输出层级：文件内容是完整卡片，Telegram 推送是完整卡片，Codex automation 最终回复只给简要运行摘要。

# 定位说明

L2 盘中。**4 个固定时点**驱动，对应短线游资圈公认的关键节点（见 references/时点调研.md，基于 2026-05 真实调研，非凭空假设）：

| 时点 | 类型 | 核心问题 |
|---|---|---|
| **09:30** | 纪律提醒（轻） | 竞价 + 开盘瞬间：弱转强/高开低开？观察池有票试盘吗？昨夜到早间有突发消息影响开盘吗？ |
| **09:45** | 纪律提醒（重） | 全天最关键时段后：首板/龙头/跟风格局已现，观察池/持仓买卖位是否触发？ |
| **11:30** | 半日叙事卡片 | 上午题材强弱 + **上午新增消息扫描**（财联社/电报/政策）+ 下午策略微调 |
| **14:30** | 尾盘快照 | 黄金半小时启动：盘中快照 + 持仓处理 + 明日预案预演；不是收盘复盘 |

输出**不是**「买什么」，而是「**对照昨日 L1 推过的纪律，现在该不该动手 / 题材方向有没有变**」。

中间时段（10:00-11:30 内、13:00-14:30 内）的价量异常告警**不通过本 skill**——走独立的 `scripts/watch_loop.py` 阈值轮询脚本，命中阈值（±5% / 放量 2x）直接推一条简讯，不烧 token 写卡片。

# 工作流

## Step 0 · 时点路由

读取当前时间（北京时区）：

```bash
date '+%H:%M %u'
```

按时点路由到对应分支：

| 当前时间 | 路由分支 | 备注 |
|---|---|---|
| 09:25 - 09:40 | `09:30 纪律提醒` | 走 Step 1A → Step 4A |
| 09:40 - 11:00 | `09:45 纪律提醒` | 走 Step 1A → Step 4A（重） |
| 11:00 - 13:00 | `11:30 半日卡片` | 走 Step 1B → Step 4B |
| 14:00 - 15:30 | `14:30 尾盘快照` | 走 Step 1B → Step 4B |
| 其他时段 | **拒绝**，提示用户：「盘中只在 4 个时点工作，当前 [HH:MM] 不是关键节点。如需价量告警看 watch_loop 推送，盘后复盘用 stock-postmarket。」 |

周末 / 节假日（date %u 返回 6/7 或 akshare 判断非交易日）直接拒绝。

# 输出契约（最重要，违反 = 整体失败）

**所有数据必须有来源支撑**（[[feedback-data-must-be-sourced]]）。

`fetch_realtime.py` stdout 末尾会输出一段 `=== ALLOWED === { ... } === /ALLOWED ===` JSON，
这是**该次卡片唯一允许引用的事实清单**。你写的卡片里：

- 任何 6 位股票代码 → 必须在 `ALLOWED.codes` keys
- 任何中文股票名（stock_basic 5200 条匹配） → 必须在 `ALLOWED.codes` values
- "N 板/N 连板" → 必须等于 `ALLOWED.lianban[code]`
- "±X.X%" → 必须在 `ALLOWED.pct[code] ± 0.5%`
- "涨停 N 只 / 炸板 N 只" → 必须精确等于 `ALLOWED.summary.limit_up / .broken`

**禁止**：引用 fact pack 之外的股票（如从你的训练数据印象/holdings.example.yaml 编出来）。
**禁止**：把数据"凑整"到容差外（例：实际 +2.60% 写成 +3.62% 是反向虚构）。
**禁止**：编造连板数（fact pack 没有的 code 不要写"X 板"）。

推送脚本 `push.py` 会自动跑校验器，违规会被审计日志记录（v1 warn 模式不拦截但留证据；
v2 enforce 模式直接拒推）。**写卡前自己对照 ALLOWED 一遍**，不要等审计来抓。

写入文件和 Telegram 推送的内容必须是卡片本身（以 ⏰/🌤️ 开头）；Codex automation 最终回复只给简要运行摘要，不要用“完成”替代文件写入和推送。

## Step 1A · 纪律提醒分支数据拉取（09:30 / 09:45）

```bash
.venv/bin/python .agents/skills/stock-intraday/scripts/fetch_realtime.py
```

输出包含：

1. **今日观察池**：优先读 `decision_tickets`，并合并手工 `/watch` 写入且买点/止损完整的 `watchlist_dynamic`；都没有时再从 `push_log` 取今日 `source='stock-premarket'` 最新一条；找不到则 fallback 读 `data/last_card.md`（mtime=今天）；都没有打印 `PREMARKET_MISSING` 警告
2. **实盘持仓**：读当前项目根目录的 `holdings.yaml`（手动维护）→ code/name/cost/shares/stop_loss/take_profit
3. **观察池+持仓的实时行情**：用 akshare `stock_zh_a_spot_em()` 拉当前价、最高、最低、涨跌幅、量比
4. **盘口快照**：每只票当前涨停价/跌停价、是否触及

读完整 stdout。

### ⚠️ 兜底：撞到 PREMARKET_MISSING（观察池为空 + stderr 出现 PREMARKET_MISSING）

可能原因：通勤无网、premarket SKILL 未跑、推送链路挂了。**此时绝对不能发"今日不参与"的反向信号**（参考 [[feedback-watchlist-vs-holding-state]]）。

按时点处理：

| 时点 | 处理 |
|---|---|
| 09:30 | 推一条**短卡**：「⏳ 盘前数据未到位（PREMARKET_MISSING）。可能因网络问题或盘前 automation 未跑。建议检查 runtime host 上的 `stock-premarket` Codex automation，必要时手动调用 `stock-premarket` 流程补生成，09:45 再判。当前空仓不动作。」**不做任何观察池/题材判断**。 |
| 09:45 | 同上短卡，措辞改为「盘前仍未到位，今日 L1 流程跳过，盯持仓即可」。**不要伪造观察池**。 |
| 11:30 / 14:30 | 走纯叙事分支（无观察池板块），明确说明今日因 PREMARKET_MISSING 无 L1 候选；其余主线/消息/持仓正常分析。 |

绝不要因为 watchlist 空就生成"今日空仓不参与追板"这种反向决策——这是策略错位。

| 09:45 | 09:00 → 现在 | 财联社早盘速递、交易所早间公告 |

2. **WebFetch** `https://www.cls.cn/depth/1003` — 早盘速递（09:30/09:45 重点用）

## Step 3 · 综合判定

### Step 3A · 纪律分支（09:30 / 09:45）

对每只决策单 + 持仓，逐只判定动作（沿用 L4 的状态机但实时版本）：

> **关键区分：决策单 = 未持仓的计划；持仓 = 已买入。两者状态机不同，禁止混用。**

#### 持仓股状态机（已买入，止损线生效）

| 状态 | 判定 | 提醒动作 |
|---|---|---|
| ✅ 持续强势 | 当前价 > 成本 + 5% 或破前高 | 派别 A：拿；派别 B/C：浮盈 ≥3% 减半 |
| ⚠️ 假突破走弱 | 已破买点后回落到买点下方 | 9:50 前破开盘价立即止损 |
| 🚨 跌破止损 | 最低 ≤ 止损价 | **立即止损**（赵老哥派纪律，不留情） |
| 💥 异动 | 盘中 -3% 或放量 2x 砸盘 | 警报：检查止损线 |

#### 决策单动作机（未买入，按 lane 裁决）

| lane | 判定 | 提醒动作 |
|---|---|---|
| `main` | 当前价进入买入区间且未超最多追价 | **买 / 等 / 放弃** 三选一，不允许模糊 |
| `ambush` | 当前价进入低吸区间 | 只提示"小仓低吸区"，涨过区间不追 |
| `backup` | 主攻作废后才触发 | 未确认主攻作废前只能等 |
| `trend` | 盘中主线确认后进入趋势买点区，未接近涨停 | 小仓趋势试错；不打板，超追价上限放弃 |
| `ban` | 任何走势 | 不买；若大涨，记录盘后复盘，不盘中追 |

#### 关键纪律（区别于赵老哥派教条）

1. **每张纪律卡必须有动作裁决**：买 / 等 / 放弃 / 卖 / 空仓，只能选这些词。
2. **未持仓不存在"止损"**：决策单的 `stop_price` 是入场后保护线；未入场前用于判断计划失效，不写成"已止损"。
3. **放弃时点默认 10:30**：主攻到 deadline 未触发就今日空仓；潜伏票按等待期，不因盘中涨起来临时追高。
4. **盘前空仓不等于全天空仓**：若 `theme_emergence_loop` 在 T1 写入自动 `decision_tickets(lane=trend)`，L2 必须把它当作趋势候选复核；满足趋势买点才允许小仓，不满足则继续观察。T2 只代表确认，不追价新增。
5. **"同板块没跟"判定要用子赛道精确匹配**，不能笼统："氟化工" ≠ "电子特气" ≠ "碳化硅" ≠ "硅光"。中船特气属电子特气；氟化工龙头先涨不代表电子特气必须同步启动（电子特气主升一般晚 30min，等氟化工先点火再传导）。判"没跟" 至少观察 30min 同子赛道动向。
6. **龙虎榜净买入 > 1 亿的票享受额外耐心**：放弃时点延后到 11:00；早盘急跌洗盘信号置信度更高。

消息面影响叠加：命中持仓的利好/利空消息单列。

### Step 3B · 叙事分支（11:30 / 14:30）

**3B.1 上午/尾盘快照定调（一段叙事，4-6 句）**

要素：
- 涨停数 vs 昨日（增减说明情绪方向）
- 最高连板梯队完整性（断板 = 顶部信号）
- 炸板数（高炸板 = 接力意愿弱）
- 主线题材是否兑现（昨日 L1 推的题材今日表现）
- 隔夜外围 / 上午突发消息的实际影响

每个数字必须带"so what"解读，禁止纯罗列。

**3B.2 题材延续性判定**（沿用 L4 3a）

对今日 reason tag / 概念板块 Top 5，每个判定 延续 / 退潮 / 新主线。

**3B.3 持仓 + 决策单现状**（逐只一行）

呼应 Step 3A 的状态机，但更精简——叙事卡片里只列关键状态变化，不堆全表。

**关键：11:30 卡片是主攻收口节点**：
- 主攻在上午触发 → 标 ✅ 已触发，下午只盯持仓处理
- 主攻到 11:30 仍未触发 → 标 ❌ 今日主攻作废，收盘前不再追
- 潜伏若只到低吸区 → 标 🟡 小仓试错；若涨过区间 → 标 🚫 已错过，不追

**3B.4 决策建议**

- 11:30 卡片：**下午策略三选一**（保持 / 减仓只盯低吸 / 直接空仓等明日）
- 14:30 卡片：**尾盘动作三选一 + 明日预演**（持仓处理 / 不动作 / 降噪等盘后）+ 3 条明早 L1 重点盯点。禁止写“收盘后决策”“全日定调”。

# 强约束

1. **时点路由是硬约束**：不在 4 个时点窗口内调用 → 拒绝执行，不要"凑合给一份"。
2. **决策单 + 持仓必须双轨**：决策单优先来自 `decision_tickets`，无决策单才回退解析旧 push_log 观察池；持仓来自 holdings.yaml（用户手动维护）。两者并行盯，不混淆。
3. **消息面是每个时点必做项**（继承 [[feedback-news-awareness]]），不是可选——题材轮动派必须看消息。
4. **纪律卡片要快、要密集**：09:30/09:45 卡片字数 ≤ 1200 字，状态机式输出，不写散文。
5. **叙事卡片要 so what**（继承 [[feedback-postmarket-style]]）：列数字必须解读；持仓处理穷举状态；下午/明日决策三选一不允许模糊措辞。
6. **假突破已破止损但用户没出**：必须在卡片里明写"应立即止损"，不要委婉。
7. **新闻必须带 URL + 时间**；24h 窗口；禁用淘股吧/股吧/微博/雪球热议。
8. **首板候选派盘中提醒**：派别 D 票如果上午同概念已现 2+ 首板，11:30 卡片要明确标"可关注盘中封板时机"。
9. 卡片长度：纪律卡 ≤ 1200 字，叙事卡 ≤ 2200 字。
10. **不在 4 时点外触发**：用户问"现在情况"如果当前不在窗口内，路由直接拒绝并解释。
11. **动作裁决必须明确**：每张卡片必须出现买 / 等 / 放弃 / 卖 / 空仓中的至少一个，不允许只写消息面和状态描述。
