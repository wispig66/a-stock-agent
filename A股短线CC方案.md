> 写作日期：2026-05-12
> 适用对象：只有普通券商 App、零基础、风格偏题材轮动的散户

## Part 0 · 把答案先给你

写这篇之前，你抛了三个问题。我把结论先放在最前面，免得你读一万字之后才发现方向不对。

**问题一：CC 能做到「百分百无干预」吗？**

不能。原因是三重否决，缺一不可：

- **合规上**：散户自动下单一旦走通，属于程序化交易，需要券商完成报告流程，且只在已开通 QMT/Ptrade 权限的散户上可行（普通券商一般要求 50 万总资产）。CC 不是券商终端，没有直连交易所的合规通道。
- **技术上**：免费行情接口是 3-6 秒快照（不是逐笔），加 LLM 推理 5-30 秒，加网络往返，端到端延迟 10-30 秒。秒级接力打板物理上做不到。
- **经济上**：盘中持续调用 Opus 一天，token 成本可能比短线那点 alpha 还高。

**问题二：那 CC 能做到什么程度？**

80% 的决策辅助 + 20% 的触发推送。最后一步（点确认下单）永远是你。

具体说：盘前给观察池、盘中给信号、盘后做复盘并制定次日策略。中间所有判断 CC 都能做，但执行回路保留人。

**问题三：你的最终推荐方案是什么？**

**「定时三段式 + 一档合规 + 飞书/PushPlus 推送」**：

- 盘前 09:25 前一次性跑完，输出今日观察池
- 盘中每 3-5 分钟轮询 + 触发式异动推送
- 盘后长会话深度复盘 + 次日策略

这套方案：合规零摩擦、月成本几十块、能落地、不依赖任何专业终端。

下面是论证过程。Part 1（30%）讲短线玩法最小必要知识——不学这部分，CC 自动化就是无源之水。Part 2（70%）讲 CC 如何落地，含可跑代码骨架。Part 3 给推荐方案和 4 周路线图。

## Part 1 · 短线玩法的最小必要知识

### 1.1 情绪周期：唯一不能跳过的心智模型

A 股短线赚的不是公司的钱，是其他散户的钱。题材轮动方向尤其如此。所以你要建模的不是公司基本面，是**群体情绪在五个阶段间的重复迁移**。

这个模型最早由淘股吧 ID「炒股养家」系统化表达。原话："短线博弈的本质是场外资金买入倾向与场内筹码卖出倾向的动态比较。"

五个阶段的判断信号、典型时长、该做什么：

| 阶段 | 关键信号 | 时长 | 该做 | 不该做 |
|------|---------|------|------|--------|
| **① 启动** | 涨停 30-60 家、2-3 连板出现、超跌或新题材打出首板赚钱效应 | 1-3 天 | 超跌首板试错、轻仓 | 追高位老票 |
| **② 加速** | 涨停 >60 家、5 板+ 高度股出现、炸板率 <30%、主线龙头明确 | 3-7 天 | 主线龙头接力、低位补涨 | 做杂毛、做非主线 |
| **③ 高位分歧** | 高度股断板或一字烂板、涨停家数回落、炸板率 30-50%、龙头次日溢价转负 | 1-3 天 | 减仓、只做高低切 | 硬抗高位、加仓接力 |
| **④ 一致**（若出现） | 分歧次日龙头放量回封、跟风同步红、市场重新一致看多 | 1 天为主 | 最后一把接力 | 误判一致后买在最后一棒 |
| **⑤ 退潮** | 涨停 <30 家、连板梯队消失、跌停 >30 家、炸板率 >50% | 3-10 天+ | 空仓为主 | 抄底老主线龙头 |

养家有句原话最值得记住："**敢于大盘低位空仓，敢于大盘高位满仓；心中无顶底，操作自随心。**"

注意这话的反直觉：低位反而空仓，高位反而满仓。因为低位 = 退潮末期，群体情绪还没修复，赚钱效应没起来，你重仓没意义；高位 = 加速期，赚钱效应自我强化，你满仓才能吃到主升。这是把「价值投资」的本能彻底反过来的思路，散户最难接受的就是这一句。

### 1.2 主流游资流派：两条对立路线

游资语录有个陷阱：散户读多了会觉得「都对」，然后混着用。结果两个流派的纪律相反，混用必爆。

**养家派（按情绪出场）**：纯情绪博弈，看赚钱/亏钱效应做仓位决策，不看基本面不看技术。核心动作是「**高手买入龙头，超级高手卖出龙头**」——真正的 alpha 在加速顶把龙头卖给一致期的接力者。养家原话里有句最危险的："永不止损，永不止盈。"——意思是出场按情绪周期，不按价格。

**赵老哥派（按价格出场）**：打板派代表，2007 年 10 万入市，2015 年自述十亿。核心理念是「**二板定龙头，一板算个毛**」——首板信号噪音太大，二板才是真龙的资金验证。只做三类首板：突破箱体、创新高、反弹首板。要求换手高、成交额 ≥3 亿。胜率约 60%，盈亏比 1.79:1，回撤 <20%。**严格止损**。

这两条路线纪律是反的。养家不止损，赵老哥严格止损。你必须选一边死磕。散户最常见的死法就是：上涨学养家「永不止盈」硬抗，下跌也学养家「永不止损」硬抗——两头被打。

**章盟主（板块派）**：1996 年入市，5 万到百亿，做整个板块而非单只票。「中短结合，趋势打板」，专做上升通道。名言："在下降通道赚钱是偶然，亏钱是必然；上升通道反之。" 本质是「只在主升期重仓」。多账户多席位协同推板块。

**作手新一（超跌反转派）**：80 后，2012 年 10 万 → 2019 年上亿。最知名打法是「超跌首板」——在退潮末期人心思涨时用超跌妖股打出赚钱效应反转。名言："妖股的逻辑就是人心思涨又找不到逻辑。"

这四派对题材轮动散户最有用的是养家（建模情绪周期）和章盟主（建模板块联动），赵老哥用来做接力纪律参考，作手新一用来理解退潮末期的反转机会。

### 1.3 打板战法的胜率结构

你不一定要打板，但必须懂打板，因为情绪周期的所有信号都从涨停结构里读出来。

- **首板**：噪音最大。但筛选后的首板（突破/创新高/反弹三类之一 + 成交额 ≥3 亿 + 换手高 + 当日唯一同主线首板）仍是金矿。
- **二板**：胜率最高的接力位置。「二板定龙头」——市场用真金验证了首板，但封单溢价还没透支。打板派的黄金位置。
- **三板**：高度兑现位，进入加速期。溢价被吃掉一截，需要题材强 + 龙头属性确立。
- **高位板（4 板+）**：进入「分歧 vs 一致」博弈区，期望收益被分歧期低开吃掉。「超级高手卖出龙头」指的就是这里。

**烂板 vs 换手板**：
- 烂板（一字 / 秒板 / 无量板）：筹码没换，封单是恐慌或惜售，次日开盘容易低开炸出
- 换手板（封单几亿、撬板再封）：筹码已经验证，主力席位接走，次日溢价稳定

短线接力**只接换手板**，烂板做 T 不接力。这是打板派共识。

### 1.4 题材发酵的六步路径

题材轮动方向的核心是把题材生命周期吃透。一个完整题材从消息到退潮经过六步，每步都有可识别的信号：

1. **消息驱动** — 政策文件 / 产业新闻 / 突发事件出来。识别信号：当日盘后或次日早盘有非主流板块龙头一字。
2. **概念识别** — 市场用一天时间筛「哪些公司沾边」。识别信号：盘中异动榜出现 3-5 只同概念集体异动，分时同向。
3. **龙头确认**（关键节点）— 二板出现，市场用真金选龙头。识别信号：同概念多只首板，但只有 1-2 只二板成功——这就是龙头。
4. **跟风扩散** — 龙头三板/四板，二线跟风首板批量出现。识别信号：题材内涨停家数从 1-2 变 5-10，赚钱效应外溢。
5. **高低切** — 龙头进入分歧，资金从高位切到低位补涨。识别信号：龙头放量滞涨或炸板，同时低位补涨股放量启动。
6. **退潮** — 龙头核按钮，跟风全线杀跌。识别信号：龙头跌停或开盘 -5% 无承接，二线同步杀跌。

记住每一步只持续 1-3 天。一个完整题材生命周期通常 5-15 天。

### 1.5 情绪指标的精确公式

这一节是 30% 玩法部分**最重要**的一节，因为 Part 2 所有 CC 自动化都建在这套指标上。每个指标都给到可代码化的口径。

| 指标 | 公式 | 阈值参考 |
|------|------|---------|
| **连板高度** | `max(当日所有连板个股的连板数)` | 2-3=启动；5+=主升；断板=分歧；消失=退潮 |
| **晋级率** | `今日连板家数 ÷ 昨日涨停家数` | >50% 强；30-50% 中；<30% 弱 |
| **二板晋级率** | `今日二板家数 ÷ 昨日首板家数` | <20% 题材不接力 |
| **炸板率** | `(当日曾涨停但收盘未封板) ÷ 当日曾涨停家数` | <30% 积极；30-50% 分歧；>50% 退潮 |
| **涨停家数** | 当日收盘封板个股数（剔除新股、ST） | >60 强；30-60 中；<30 弱 |
| **跌停家数** | 当日收盘跌停个股数（剔除 ST） | <10 安全；10-30 分歧；>30 退潮 |
| **打板赚钱效应** | `mean(昨涨停股今日开盘溢价%)` | ≥3% 强；-3%~3% 分歧；≤-5% 退潮 |
| **亏钱效应** | `count(昨涨停今低开≥5% 且无承接)` | >0 即退潮信号 |
| **低吸赚钱效应** | 主线低位个股能否红盘出局 | 能红盘=情绪正常；低吸也亏=退潮确认 |
| **上证连板指** | 所有连板个股等权指数（韭研公社/同花顺有现成口径） | 趋势线下穿=情绪转弱 |

把这十个指标每天落库，三个月之后你就有了情绪周期阶段判定的本地数据。CC 在盘后只要查一遍今天的指标值，对照阈值表，就能输出「今天处于哪个阶段、明天该怎么做」的结构化判断。这是整套方案的核心引擎。

### 1.6 龙虎榜的真相

龙虎榜是 A 股**唯一**事后公开的大资金动向数据。涨幅榜或成交榜上榜的票，T+1 公告买卖前五席位金额。

但请记住一句话：**龙虎榜是事后数据，不是事前信号。**

T+1 公告 = 永远晚一天。所谓「跟游资」在执行层面不可能跟买入价，只能跟方向，而方向往往次日已被透支。

那龙虎榜的真正价值在哪？

- **识别游资席位风格**：赵老哥喜欢突破首板、章盟主板块联动、作手新一超跌首板——识别席位 ≈ 识别打法
- **验证主线**：多游资席位协同上榜 + 机构同向 = 高胜率信号
- **反向指标**：散户营业部（如东财拉萨团结路二/东环路二）大量出现在买方席位 = 后续大概率跌

顶级席位清单（写代码时识别用）：

- 东财拉萨团结路第二、东环路第二——「拉萨天团」，免佣金散户 + 部分游资混杂
- 华泰深圳益田路——纯打板敢死队
- 国泰君安上海江苏路——老牌游资大本营
- 国泰君安南京太平南路——作手新一旧旗舰
- 银河证券绍兴——通信、汽车、非银

这部分到此为止。如果你想更深地学短线，去读养家原帖、赵老哥淘股吧实盘日志。但写代码、做 CC 自动化、跑出第一个能用的系统——上面这些已经够了。

## Part 2 · CC 能做到什么（核心）

### 2.1 先把边界讲清楚

不诚实地讲边界就是骗人。三层边界叠加：

**物理边界**

免费实时行情接口（新浪 hq.sinajs.cn、东财 push2.eastmoney.com、腾讯 qt.gtimg.cn）都是 snapshot，不是 tick。刷新间隔 3-6 秒。加 Python 处理 ~1 秒，加 LLM 推理 5-30 秒，加你从手机看到信号到下单 5-10 秒。**端到端延迟 10-30 秒。**

这意味着：

- 秒级接力打板：做不到
- 抢首板封单：做不到（首板封单常在 0.5 秒内堆完）
- 分钟级题材监控、自选股异动报警、接力候选名单维护：完全够用
- 盘后复盘：完全够用，且是 CC 最划算的环节

要做秒级打板，最低成本是 L2 行情（同花顺/通达信 L2 ≈ ¥1500/年），但 L2 仍不能下单，下单要 QMT/Ptrade（散户开通门槛一般是券商总资产 50 万）。这条路线对零基础用户不推荐。

**合规边界**

证监会 [2024] 8 号《证券市场程序化交易管理规定（试行）》2024-10-08 施行；沪深北实施细则 2025-07-07 起统一施行。「程序化交易」的定义是「**通过计算机程序自动生成或者下达交易指令**」。

散户三档自动化的合规判断：

| 档 | 形态 | 是否构成程序化交易 | 是否合规 | 报告义务 |
|---|---|---|---|---|
| 一档 | CC 选股 + 手动下单 | 否 | 完全合规 | 无 |
| 二档 | QMT/Ptrade 自动下单（哪怕一天一笔） | 是 | 合规（需走流程） | 经券商报告账户/资金/策略/软件 |
| 三档 | 触发每秒 300 笔或全日 20000 笔 | 是，且属高频 | 合规但严管 | 额外报告服务器/测试/应急；散户实质上几乎无法触及 |

上交所答记者问明确："中小个人投资者基本未发生相关情况，正常交易行为不会受影响。"

**核心结论**：本文推荐的一档（CC 给信号 + 你手动下单）**没有任何合规摩擦**。这也是为什么这套方案值得做。

灰区警告：用 Python 模拟点击券商 APP（按键精灵类）做「半自动」，严格说属第二档，需经券商报告。别图省事踩这个坑。

**经济边界**

盘中如果让 CC 连续盯盘 4 小时，每分钟一次推理，每次 5-15 秒 + 几千 tokens——一天下来 token 成本可能就是几十到上百块。一个月几千。短线 alpha 抵不抵得过？大概率抵不过。

正确做法是：**让 LLM 只在必要的判断时刻被唤醒**，不要让它持续运行。这是 2.2 节架构的根本动机。

### 2.2 推荐架构：定时三段式

整体架构图（文字版）：

```
盘前 08:30-09:25
└─ launchd (macOS) / cron (Linux)
   └─ claude --bare -p "today_premarket.prompt"
      ├─ WebFetch: 财经新闻 / 政策公告
      ├─ MCP: akshare / tushare 拉昨日龙虎榜 + 涨停池
      ├─ MCP: pywencai 跑 5 条自然语言选股
      └─ MCP: 飞书 webhook 推送观察池

盘中 09:30-15:00
├─ 路径 A：cron 每 3-5 分钟
│  └─ claude -p "intraday_scan.prompt"
│     ├─ 扫描观察池（涨跌幅、量比、突破均线）
│     ├─ 监控概念板块涨幅榜变化（题材轮动）
│     └─ 命中则飞书推送
│
└─ 路径 B：本地常驻盯盘脚本（纯 Python，无 LLM）
   └─ 命中条件 → 触发独立 claude -p 做二次确认 + 推送

盘后 15:00-次日
└─ 手动启动长会话（或 cron 在 15:30 起）
   └─ claude --continue 或 claude --resume <session_id>
      ├─ 解析当日成交 + 龙虎榜
      ├─ 计算今日情绪指标十件套
      ├─ 入库 (SQLite)
      ├─ 情绪周期阶段判定
      ├─ 输出明日策略 + 仓位建议
      └─ PushPlus 推送复盘到微信
```

三个关键设计决策：

1. **盘中不持续运行 LLM**：cron 短任务，每次跑完即退。避免内存泄漏、避免 token 烧穿、避免 API 偶发限流影响后续。
2. **触发式 + 定时式并存**：定时保覆盖率，触发保实时性。完全靠定时会错过秒级异动，完全靠触发会漏掉缓慢变化。
3. **本地优先，不上云服务器**：散户用云服务器是过度工程。你的 MacBook 或一台旧 mini PC 留着开机，比你折腾 AWS/阿里云便宜也稳。

### 2.3 数据栈搭建（从零开始的 30 分钟教程）

零基础先把环境搞起来。

**Step 1：装 Python 和包**

```bash
# macOS（推荐 brew + pyenv，全程零冲突）
brew install pyenv node
pyenv install 3.11.7
pyenv global 3.11.7

# 建项目目录
mkdir -p ~/Desktop/stock/{code,data,logs,prompts}
cd ~/Desktop/stock

# 装核心包
pip install akshare efinance pywencai pandas pyarrow duckdb requests
```

注意 `pywencai` 需要 Node.js v16+（要跑 hexin-v 的 JS 生成动态参数）。`brew install node` 已经搞定。

**Step 2：三档预算方案**

建表 SQL（一次性跑完即可）：

```sql
CREATE TABLE IF NOT EXISTS daily_kline (
    code TEXT, date TEXT,
    open REAL, high REAL, low REAL, close REAL,
    vol REAL, amount REAL,
    PRIMARY KEY (code, date)
);

CREATE TABLE IF NOT EXISTS limit_up (
    date TEXT, code TEXT, name TEXT,
    limit_up_num INTEGER,         -- 连板数
    seal_amount REAL,              -- 封单金额
    turnover_rate REAL,            -- 换手率
    first_seal_time TEXT,          -- 首次封板时间
    open_count INTEGER,            -- 开板次数
    concept TEXT,                  -- 题材
    PRIMARY KEY (date, code)
);

CREATE TABLE IF NOT EXISTS lhb (
    date TEXT, code TEXT, name TEXT,
    seat_name TEXT,                -- 营业部名称
    buy_amount REAL, sell_amount REAL, net_amount REAL,
    rank INTEGER                   -- 1-5 买方 / 6-10 卖方
);

CREATE TABLE IF NOT EXISTS sentiment_daily (
    date TEXT PRIMARY KEY,
    limit_up_count INTEGER,
    limit_down_count INTEGER,
    max_consec INTEGER,             -- 连板高度
    promotion_rate REAL,            -- 晋级率
    second_promotion_rate REAL,     -- 二板晋级率
    blast_rate REAL,                -- 炸板率
    money_effect REAL,              -- 打板赚钱效应（昨涨停今溢价均值）
    loss_effect INTEGER,            -- 亏钱效应（昨涨停今低开≥5%家数）
    phase TEXT                       -- 启动/加速/分歧/一致/退潮
);
```

这套表三个月之后你就有了完整的情绪周期数据库，CC 可以直接 SQL 查询做判断。

### 2.5 盘中模块：分钟级扫描 + 触发式信号

**路径 A：cron 每 3 分钟扫描**

`~/Desktop/stock/code/intraday_scan.py`（这是给 CC 用的，但纯 Python 也能跑，先跑通再让 CC 加智能）：

```python
import efinance as ef
import json
from datetime import datetime
import requests

WATCHLIST_PATH = "~/Desktop/stock/data/today_watchlist.json"
FEISHU_WEBHOOK = "https://open.feishu.cn/open-apis/bot/v2/hook/YOUR_TOKEN"

def push_feishu(text: str):
    requests.post(FEISHU_WEBHOOK,
        json={"msg_type": "text", "content": {"text": text}})

def main():
    # TODO: 加载今日观察池（盘前模块输出）
    watchlist = json.load(open(WATCHLIST_PATH))
    codes = [item['code'] for item in watchlist]

    # 拉实时快照
    df = ef.stock.get_realtime_quotes()  # 全市场，约 5000 行
    df = df[df['代码'].isin(codes)]

    signals = []
    for _, row in df.iterrows():
        code, name = row['代码'], row['名称']
        pct = row['涨跌幅']
        vol_ratio = row['量比']

        # TODO: 这里把 1.5 节的指标拆细：
        #   - 突破昨日涨停价
        #   - 量比 > 2 且涨幅 > 5%
        #   - 跌破支撑位（止损提醒）
        if pct >= 7 and vol_ratio >= 2:
            signals.append(f"⚡ {code} {name} 涨 {pct}% 量比 {vol_ratio}")

    if signals:
        push_feishu(f"[{datetime.now():%H:%M}] 盘中异动\n" + "\n".join(signals))

if __name__ == "__main__":
    main()
```

cron 配置（`crontab -e`）：

```

*/3 13-14 * * 1-5 cd ~/Desktop/stock && python code/intraday_scan.py
```

注意时间段错开开盘前和午休。9:30-11:30 + 13:00-15:00 是交易时段。

**路径 B：触发式调用 CC**

当 Path A 检测到「重大异动」（连板、放量突破、龙头炸板），用 CC 做二次智能判断。

```python
# 在 intraday_scan.py 末尾追加
import subprocess

def trigger_cc_deep_analysis(code: str, signal: str):
    """对单个重大异动调用 CC 做深度分析"""
    prompt = f"""
当前时间 {datetime.now():%H:%M}。{code} 触发 {signal}。

请你：
1. 用 akshare 拉这只票今日分时图（mins=1）
2. 查它属于哪些概念板块，今日板块涨幅排名如何
3. 查它最近 5 个交易日的涨停结构（是几板？前几板封单大小？换手率？）
4. 结合当前情绪周期阶段（查 sentiment_daily 最新一条），判断：
   - 这是接力机会、止盈位、还是诱多陷阱？
   - 关键位是多少？跌破多少需要止损/止盈？
5. 输出 100 字以内的飞书卡片消息，含明确动作建议。
"""
    # claude -p 模式，30 秒超时
    result = subprocess.run(
        ["claude", "-p", prompt, "--output-format", "text"],
        capture_output=True, text=True, timeout=60
    )
    push_feishu_card(result.stdout)
```

**为什么这套架构最划算**：纯 Python 扫描 5000 只票只要 1-2 秒、零成本；只有遇到真正需要判断的票（一天 5-20 次）才调用 CC。token 成本可控在每日几块钱以内。

**明确不做的事**：

- 不做秒级追板（物理做不到）
- 不做首板抢筹（首板封单 0.5 秒内堆完，免费快照看不到）
- 不做毫秒级量化（要 colocation + L2，散户不可能）

### 2.6 盘后模块：复盘 + 次日策略

CC **最划算**的环节。每天 15:30 启动，长会话，深度分析。

**核心 prompt**（`~/Desktop/stock/prompts/postmarket.md`）：

```
今天是 {date}，A 股已收盘。请按以下流程做完整复盘：

# Step 1：数据采集
- akshare 拉今日全市场涨停池 → 入库 limit_up 表
- akshare 拉今日龙虎榜 → 入库 lhb 表
- 标记游资席位（参考 seat_lookup 表的顶级席位列表）

# Step 2：情绪指标计算（写入 sentiment_daily 表）
按 1.5 节公式计算：
- limit_up_count（涨停家数）
- limit_down_count（跌停家数）
- max_consec（连板高度）
- promotion_rate = 今日连板家数 / 昨日涨停家数
- second_promotion_rate = 今日二板家数 / 昨日首板家数
- blast_rate = 炸板家数 / (炸板+涨停)
- money_effect = mean(昨涨停股今开盘溢价%)
- loss_effect = count(昨涨停今低开>=5% 且无承接)

# Step 3：情绪周期阶段判定
对照 1.1 节五阶段阈值表，给出今日阶段判定。
若与昨日不同，标明"阶段切换"。

# Step 4：主线识别
- 今日涨停最多的 3 个概念
- 每个概念的龙头股（连板最高的那只）
- 龙头股的龙虎榜资金性质（游资 / 机构 / 散户）

# Step 5：持仓与未执行信号复盘
- 我今日的持仓（从 positions.json 读）：盈亏、是否触及止盈止损线
- 今日盘中推送过的信号：若执行了，结果如何；若没执行，事后看该不该执行

# Step 6：次日策略
基于情绪阶段，输出：
- 仓位建议（0% / 30% / 60% / 80%）
- 重点关注题材（最多 3 个）
- 接力候选名单（最多 5 只，含理由）
- 风险点

# Step 7：PushPlus 推送复盘到微信
调 push_plus MCP，长卡片格式。
```

**为什么用 `--resume`**：盘后是长会话，保留前一日策略上下文，CC 才能看到「昨日推送了什么 / 今日实际发生什么」的对比，复盘才有连续性。第一次跑用 `--continue` 或新会话，把 `session_id` 写到 `.session_id` 文件。

```python
from time import sleep
def with_retry(fn, retries=3, base_delay=2):
    for i in range(retries):
        try: return fn()
        except Exception as e:
            if i == retries - 1: raise
            sleep(base_delay * (2 ** i))  # 指数退避
```

## Part 3 · 最终推荐方案

### 3.1 一句话定义

**「三段式决策辅助」**：

- CC = 研究员 + 操盘助理
- 你 = 最终决策人 + 唯一下单人
- 盘前给你观察池、盘中给你信号、盘后给你复盘
- 不直连交易、不自动下单、不追求秒级
- 合规零摩擦、月成本可控、每天投入 30-60 分钟

**适用人群**：

- 想做 A 股短线（题材轮动方向）的散户
- 工作日 09:15、12:00、15:30 三个时间点能看手机的人
- 接受前期 1-3 个月数据积累期没有 alpha、只为建立体系
- 能接受短线本质是高风险高换手的人

**不适用人群**：

- 想找「全自动印钞机」的人——本文已经诚实告诉你不存在
- 资金大到值得申请 QMT/Ptrade（≥50 万）的人——你的方案应该是另一篇文章
- 完全不能看盘的人（出差、上课、医院）——短线对你不友好，做中长线更合理

- 加触发式 CC 深度分析（连板、突破、龙头炸板三类信号）
- cron 配置 9:30-11:30 / 13:00-15:00 每 3 分钟跑
- 验收：连续 5 个交易日，盘中能收到 5-15 条飞书消息，每条都能看懂
- 时间投入：周末两个全天

**Week 5 之后**：开始用、改 prompt、改阈值。你会发现 70% 的工作是调 prompt 和阈值，不是写代码。这正常。

### 3.3 一年后你拿到的是什么

**会拿到**：

- 一份完整的个人情绪周期数据库（250+ 个交易日的情绪指标）
- 一套被你反复调优的 prompt 库（盘前/盘中/盘后各几个）
- 一份策略日志 git 仓库（每天的判断 + 实际结果对比）
- 一套对短线情绪周期的本能判断力——这是最值钱的，因为是经过实战的
- 一份对「自己适不适合做短线」的诚实答案

**不会拿到**：

- 暴富（说想暴富的请去赌场，效率高得多）
- 稳定盈利的「圣杯策略」（不存在，市场是非平稳的）
- 夏普 2 的策略（机构都做不到，何况零基础散户用免费数据）
- 「完全无人值守自动赚钱」（参见 Part 0 的三重否决）

**真实预期收益**：

- 如果你勤奋、纪律严、能接受亏损：第一年大概率小亏到小赚（-10% 到 +10%）
- 第二年开始可能稳定打平或微利
- 跑赢沪深 300 是合理目标，跑赢游资大佬是不合理目标

### 3.4 常见踩坑清单

- pywencai 高频被封 cookie → 只在盘前一次性调，盘中绝不调
- pywencai cookie 过期 → 每周固定一天检查并更新（写个验活脚本）
- Node 没装就装 pywencai → 报错神奇，先 `brew install node`
- 飞书 webhook 5 次/秒上限 → 批量信号要合并，不要 for 循环 push
- 把回测当实盘 → 回测永远过拟合，先实盘 1 个月再信任任何策略
- 跨账户搬交易 → 别折腾，一个账户跑透
- 不写日志 → 三个月后你完全不记得当时为什么买，等于白做
- 让 LLM 持续盯盘 → token 烧穿，参见 2.1 经济边界
- 用 Python 模拟点击券商 APP → 灰区，参见 2.1 合规边界
- 把 CC 输出直接当真 → CC 会幻觉，所有数字（价格、家数）必须用 MCP 调真实数据源，不要让 LLM 凭记忆生成
- 第一周就想做盘中触发 → 先把盘后复盘跑稳，再做盘中，否则容易因为盘中信号情绪化操作
