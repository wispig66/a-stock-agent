# 散户三档自动化的合规分级
依据《证券市场程序化交易管理规定（试行）》及沪深交易所实施细则
信息截止: 2026-05-12

## 第一档：AI 辅助决策 + 人工下单（Claude Code 用例）
- 指令既非自动生成也非自动下达 → **不构成程序化交易**
- 不在管理规定的适用范围
- 无报告义务、无审批、无限制
- 合规性：完全合规

## 第二档：QMT/Ptrade 等量化客户端半自动下单
- 指令由程序自动下达 → **构成程序化交易**
- 个人投资者通过券商 QMT 等客户端做策略自动下单，需通过券商完成报告：账户、资金、策略、软件信息
- 一般券商对个人开通 QMT 有门槛（如金融资产 50 万 + 两年交易经验等，类似科创板门槛）
- 合规性：依规报告即可

## 第三档：高频或机构级程序化
- 触发任一阈值（每秒 300 笔或全日 20000 笔）即定性为高频
- 额外报告服务器位置、系统测试、应急方案
- 适用更严的异常交易监控、可能加收差异化费用
- 个人投资者实质上极难触及此阈值；如果触及，监管会以异常交易处置

## 总结表
| 档位 | 是否程序化 | 报告 | 适用人群 |
|---|---|---|---|
| AI 辅助 + 人工点单 | 否 | 不需 | 全员适用 |
| QMT 半自动 | 是 | 经券商报告 | 已开通量化权限的散户 |
| 高频 | 是（高频类） | 高频额外报告 | 实质上几乎只有机构 |

## 容易踩雷的灰区
- 用 Python + 券商接口直连自动下单（无论速率多低）→ 已属第二档，需要报告
- AI 给出信号后用 Python 模拟点击券商 APP（按键精灵类）→ 严格说，"自动下达"成立，属第二档
- "AI 给信号自动推送给我，我看到后自己手点" → 第一档，纯辅助
