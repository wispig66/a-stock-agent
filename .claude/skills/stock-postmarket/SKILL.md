# 定位说明

L4 盘后复盘。**两个核心职责**：

1. **数据落库**：今日 sentiment 写 `sentiment_daily` 表（明日 L1 阶段判定有 10 日基线）+ 同花顺 reason tags 写 `ths_hot_reason` 表（积累题材轮动历史）
2. **复盘 + 明日预案**：对照今日 fact pack 和上午推送的盘前观察池（push_log 表里 msg_id），评估今日主线延续性，给出明日「再战 / 减仓 / 空仓」决策

输出**不是**「今日哪只赚了」，而是「**今日情绪给明日多少能见度 + 哪个主线值得明早盯**」。

# 工作流

每次调用按序执行下面 4 步。

## Step 1 · 拉今日完整数据 + 写库

```bash
~/miniconda3/envs/stock/bin/python .claude/skills/stock-postmarket/scripts/fetch_postmarket.py
```

这一步同时做三件事：

1. 从 akshare + 同花顺拉今日完整收盘数据
2. 计算 phase + 写 `sentiment_daily`（UPSERT，重跑无害）
3. 写 `ths_hot_reason`（含完整 reason tag 个股清单）
4. 渲染 `data/fact_pack/YYYYMMDD_postmarket.md`

读完整 stdout。这是今日 fact pack（6 节）：今日情绪快照 / 连板梯队 / 跌停炸板 / 龙虎榜 / 同花顺题材 Top 10 / 近 10 日情绪表。

## Step 2 · 复盘今日盘前观察池

从 `push_log` 取今日 `source='stock-premarket'` 的最新一条推送（即今早 08:30 的观察池），逐只对照今日实际行情：

```bash
# 1) 找到今日盘前推送
sqlite3 data/daily.db "SELECT msg_id, text FROM push_log
  WHERE source='stock-premarket'
    AND date(timestamp) = date('now', 'localtime')
  ORDER BY id DESC LIMIT 1;"

# 2) 对每只候选，查今日实际收盘 / 最高 / 最低
~/miniconda3/envs/stock/bin/python -c "
import akshare as ak
df = ak.stock_zh_a_spot_em()
codes = ['601991', '600076', '600683', '600869', '300449', '000601', '000925']  # 替换为今日观察池实际代码
print(df[df['代码'].isin(codes)][['代码','名称','最新价','最高','最低','涨跌幅','换手率']].to_string(index=False))
"
```

对每只候选判定：

| 状态 | 判定标准 | 含义 |
|------|---------|------|
| ✅ 触发 + 收红 | 盘中最高 ≥ 买点 且 收盘 > 买点 | 派别 A 真接力，已锁仓 |
| ⚠️ 触发 + 收绿 | 盘中最高 ≥ 买点 但 收盘 < 买点 | 假突破 / 上影线，需止损 |
| ❌ 未触发 | 盘中最高 < 买点 | 9:50 已放弃，无操作 |
| 💥 跌破止损 | 最低 ≤ 止损价 | 必须已止损出，回看竞价或盘中迟疑要扣分 |

**输出复盘表**（每只一行）+ **命中率统计**（触发率 / 收红率 / 假突破率 / 止损命中率）。

## Step 3 · 综合判定明日主线

读完 Step 1 fact pack 的「五、同花顺题材标签 Top 10」+「四、龙虎榜上榜原因」+「六、近 10 日情绪表」，判断：

### 3a. 题材延续性

对今日 reason tag Top 10 每个题材：

- **延续**：今日该题材 ≥3 只涨停 且 龙虎榜净买入 Top 5 至少有 1 只属于该题材
- **退潮**：今日该题材 ≤1 只涨停，或核心龙头炸板/跌幅 > 5%
- **新主线**：今日 reason 出现次数比昨日翻倍 且 有 ≥3 只联板（连板数 ≥ 2）

### 3b. 情绪阶段切换预判

对照「近 10 日情绪表」最近 3 天：

- 涨停连续 > 60 三天 + 今日炸板率上升 → **明日大概率高位分歧**
- 涨停 < 30 + 跌停 > 30 → **明日延续退潮，建议空仓**
- 涨停从 < 40 跳到 > 60 + 高度梯队补齐 → **明日加速可能**

### 3c. 明日决策（三选一）

| 决策 | 触发条件 | 仓位建议 |
|------|---------|---------|
| **再战** | 阶段稳定在加速 + 主线题材延续 | 总仓位维持，重点跟新主线 |
| **减仓观望** | 高位分歧信号 / 主线退潮 / 触发率 < 50% | 砍 50% 仓位，明早只盯低吸 |
| **空仓** | 退潮 / 跌停 > 30 / 龙头崩盘 | 不接力，等下一波启动信号 |

## Step 4 · 输出 Telegram 卡片

**这张卡片是写给"明天早上要做决策的人"看的，不是数据看板。** 用人话讲今天市场为什么这么走、持仓怎么处理、明天该盯什么。

格式约束（重要）：
- 推送会自动 md→Telegram HTML，**用 `**bold**` 标重点**，不要用 `#` 标题（在 Telegram 会被转粗体但不如直接 emoji+bold 干净）。
- **禁止 markdown 表格**（虽然转换器能转 bullet，但叙事场景下表格会割裂阅读）。用编号列表或缩进。
- 不要堆数字，每个数字要解释"意味着什么"。

模板：

```
🌆 <b>盘后复盘 · [YYYY-MM-DD]</b>

📖 <b>今日市场怎么走</b>

[一段 4-6 句的叙事，必须解释因果链，不是罗列数据]

举例参考写法：
"今天主线还是电力 / 算电协同，大唐发电 5 板封死，但 3 板梯队（康欣 / 京投）盘中开板，
说明承接乏力。龙虎榜算电协同共净买入 X 亿，机构资金还在；但散户接力意愿弱（炸板 23 只，
百邦炸 38 次说明顶部抛压重）。情绪从加速切换到加速末段-分歧前夜。隔夜美股 -0.66% 偏空。"

要点：
- 引用 1-2 个具体数字（涨停数 / 最高连板 / 净买额）但每个都要带解读
- 把今天 vs 昨天的"变化方向"说清楚（资金流向 / 高度梯队 / 题材轮动）
- 一句话定调今日情绪（什么阶段 + 偏多还是偏空）

🎯 <b>持仓处理</b>（无持仓写"今日空仓，跳过"）

如果上午 L1 推过观察池且**用户买入了**：

[逐只用一段话讲清楚明早怎么办]

- <b>601991 大唐发电</b>（假设买入价 6.77，今收 ?.??）
  状态：[已触发 + 收红 / 已触发 + 收绿假突破 / 未触发未持仓 / 已止损]
  明早动作：[具体三选一]
  · 浮盈 ≥3%：竞价 +3% 减半，剩余拿到破 5 日线清仓
  · 浮亏 < 止损：开盘看分时，破开盘价立即出
  · 已假突破：今日盘中就该止损，未止损则明早第一时间市价出
  · 阴线但未破止损：明早 9:50 前再观察，破 5 日线出

⚠️ <b>核心规则</b>：买入未止损的，明早开盘 5 分钟内必处理；不要让昨日错过的止损拖到今天。

🔥 <b>主线观察</b>

逐个主线题材给"延续 / 退潮 / 新主线"判断 + 明日动作：

✅ <b>[题材 A]</b>：今日 [现象] → 明日 [动作建议]
  例："算电协同：今日 6 只涨停 + 龙虎榜净买 8 亿 + 大唐封死 → 明日开盘 1 小时盯大唐能否新高"

🔻 <b>[题材 B]</b>：今日 [退潮信号] → 不参与
  例："光伏：今日仅 1 只涨停且龙头炸板 → 不进观察池"

🆕 <b>[题材 C]</b>：今日 [新信号] → 关注 1-2 个交易日
  例："商业航天：reason tag 上升中（远东 / 众合 / 通光），但还无 3 板龙头 → 关注"

🎯 <b>明日决策：[再战 / 减仓观望 / 空仓]</b>

理由 2-3 行，必须呼应"今日市场怎么走"那段的因果：

- [呼应高度梯队 / 龙头表现]
- [呼应炸板率 / 跌停结构]
- [呼应外围 / 已知事件]

仓位指引：
- 再战：维持 50-70% 总仓位
- 减仓观望：砍到 30-40%，只留确定性强的低吸
- 空仓：清掉，等下一波启动信号

📌 <b>明早 L1 重点盯什么</b>

3-5 条具体可执行的盯点（必须可观察 / 可量化）：

1. <b>[标的代码 名称]</b> 能否守 [关键价位]
   破则 [意味什么]，守住则 [意味什么]
   例："大唐发电 6.70 能否守住，破则主线退潮，守住则继续观察 6.90 压力位"

2. <b>[题材] reason tag</b> 是否继续高频出现（看明早 L1 fact pack 5.1）

3. <b>[关键事件 / 数据]</b>：[时点] [影响]
   例："09:30 集合竞价开盘价 vs 昨收，高开 >2% 多则乐观，低开则跟昨日盘中弱势一致"

⚠️ <b>风险提示</b>

2 条，**必须基于今日数据**，不要套话：

- [基于今日炸板池 / 高度断板 / 龙虎榜异常的具体风险]
- [基于明日已知催化 / 隔夜外围的风险]

——————————
本系统纪律：盘后必复盘 · 持仓必处理 · 9:50 纪律严守
明早 08:30 L1 自动接力盯盘
fact pack: data/fact_pack/[YYYYMMDD]_postmarket.md
```

**写作守则**：

1. **每一段都要回答"so what"**：列了数字必须解读，不解读不写。
2. **持仓处理段的 5 个分支必须穷举**：已触发收红 / 触发假突破 / 未触发 / 已止损 / 阴线未破止损。即使用户没透露持仓，按"假设买了"语气给指引。
3. **明日决策三选一，不允许模糊**："再战减仓"这种说法直接重写。
4. **3-5 条明早盯点必须可量化**：禁止"关注算力板块"这种空话，要写"工业富联 30 亿封单是否守住"。

# 强约束

1. **复盘必须基于今日实际行情**，不能编。akshare `stock_zh_a_spot_em()` 是收盘后真实数据。
2. **假突破必须计入命中率分母**：触发 ≠ 成功，收红才算成功。
3. **题材延续判定必须双重验证**：涨停数 + 龙虎榜资金至少一项支持，不能纯凭 reason tag 数量。
4. **明日决策三选一**，不允许"再战减仓"这种模糊措辞。
5. **空仓也是决策**，不要为给建议而强行给建议。
6. 卡片 ≤ 2200 字（盘后卡片可以略长，因为复盘需要逐只列）
7. **赵老哥派纪律**：假突破已破止损但你忘记止损 / 盘中迟疑没出 → 写明，作为下次教训
