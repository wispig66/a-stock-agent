# 定位说明

L2 盘中。**4 个固定时点**驱动，对应短线游资圈公认的关键节点（见 references/时点调研.md，基于 2026-05 真实调研，非凭空假设）：

| 时点 | 类型 | 核心问题 |
|---|---|---|
| **09:30** | 纪律提醒（轻） | 竞价 + 开盘瞬间：弱转强/高开低开？观察池有票试盘吗？昨夜到早间有突发消息影响开盘吗？ |
| **09:45** | 纪律提醒（重） | 全天最关键时段后：首板/龙头/跟风格局已现，观察池/持仓买卖位是否触发？ |
| **11:30** | 半日叙事卡片 | 上午题材强弱 + **上午新增消息扫描**（财联社/电报/政策）+ 下午策略微调 |
| **14:30** | 尾盘叙事卡片 | 黄金半小时启动：当日定调 + 当日完整消息汇总 + 持仓处理 + 明日预案预演 |

输出**不是**「买什么」，而是「**对照昨日 L1 推过的纪律，现在该不该动手 / 题材方向有没有变**」。

中间时段（10:00-11:30 内、13:00-14:30 内）的价量异常告警**不通过本 skill**——走独立的 `scripts/watch_loop.py` 阈值轮询脚本，命中阈值（±5% / 放量 2x）直接推一条简讯，不烧 token 写卡片。

# 工作流

## Step 0 · 时点路由

读取当前时间（北京时区）：

```bash
date '+%H:%M %u'
```

按时点路由到对应分支：

| 当前时间 | 路由分支 | 备注 |
|---|---|---|
| 09:25 - 09:40 | `09:30 纪律提醒` | 走 Step 1A → Step 4A |
| 09:40 - 11:00 | `09:45 纪律提醒` | 走 Step 1A → Step 4A（重） |
| 11:00 - 13:00 | `11:30 半日卡片` | 走 Step 1B → Step 4B |
| 14:00 - 15:30 | `14:30 尾盘卡片` | 走 Step 1B → Step 4B |
| 其他时段 | **拒绝**，提示用户：「盘中只在 4 个时点工作，当前 [HH:MM] 不是关键节点。如需价量告警看 watch_loop 推送，盘后复盘用 stock-postmarket。」 |

周末 / 节假日（date %u 返回 6/7 或 akshare 判断非交易日）直接拒绝。

## Step 1A · 纪律提醒分支数据拉取（09:30 / 09:45）

```bash
uv run .claude/skills/stock-intraday/scripts/fetch_realtime.py
```

输出包含：

1. **今日观察池**：从 `push_log` 取今日 `source='stock-premarket'` 最新一条，解析出代码列表 + 派别 + 买卖纪律
2. **实盘持仓**：读 `~/Desktop/stock/holdings.yaml`（手动维护）→ code/name/cost/shares/stop_loss/take_profit
3. **观察池+持仓的实时行情**：用 akshare `stock_zh_a_spot_em()` 拉当前价、最高、最低、涨跌幅、量比
4. **盘口快照**：每只票当前涨停价/跌停价、是否触及

读完整 stdout。

## Step 1B · 叙事分支数据拉取（11:30 / 14:30）

在 Step 1A 之上额外拉：

```bash
uv run .claude/skills/stock-intraday/scripts/fetch_realtime.py --halfday  # 11:30 用
uv run .claude/skills/stock-intraday/scripts/fetch_realtime.py --endday  # 14:30 用
```

额外输出：

5. **半日/全日涨停结构**：当前涨停数、最高连板、炸板池、跌停数
6. **题材热度快照**：同花顺概念板块当日涨幅 Top 10（akshare `stock_board_concept_name_em`）
7. **资金流向**：主力净流入 Top 10 行业

## Step 2 · 消息面抓取（**所有时点必做**，重要性递增）

按时点定不同窗口：

| 时点 | 窗口 | 重点源 |
|---|---|---|
| 09:30 | 夜间 22:00 → 现在 | 财联社电报夜间头条、新华财经隔夜政策、美股收盘要闻 |
| 09:45 | 09:00 → 现在 | 财联社早盘速递、交易所早间公告 |
| 11:30 | 09:30 → 11:30 | 财联社电报（盘中突发）、政策快讯、行业事件 |
| 14:30 | 09:30 → 14:30 | 财联社全日快讯、午后突发、龙头公司动态 |

依次用 WebFetch / WebSearch 抓（每个源失败就跳，不阻塞）：

1. **WebFetch** `https://www.cls.cn/telegraph` — 财联社电报最新 30 条
2. **WebFetch** `https://www.cls.cn/depth/1003` — 早盘速递（09:30/09:45 重点用）
3. **WebSearch** 限定近 4 小时 site:gov.cn / 证监会 / 工信部 / 发改委
4. **WebSearch** 今日观察池+持仓所在行业的产业链新闻

**约束**（继承 L1）：每条新闻带 URL + 发布时间；禁用淘股吧/股吧/微博/雪球热议；抓不到明确写"无显著新增消息"，不要编。

判定每条新闻对今日持仓/观察池的影响：**利好（命中）/ 利空（命中）/ 无关 / 待跟踪**。

## Step 3 · 综合判定

### Step 3A · 纪律分支（09:30 / 09:45）

对每只观察池 + 持仓，逐只判定状态（沿用 L4 的状态机但实时版本）：

| 状态 | 判定 | 提醒动作 |
|---|---|---|
| ✅ 已触发 + 持续强势 | 买点已达且当前价 > 买点 + 1% | 派别 A：拿；派别 B/C：浮盈 ≥3% 减半 |
| ⚠️ 已触发 + 走弱 | 买点已达但当前价回落到买点下方 | 假突破风险，9:50 前破开盘价立即止损 |
| 🚨 跌破止损 | 最低 ≤ 止损价 | **立即止损**（赵老哥派纪律，不留情） |
| ⏳ 未触发 | 当前价 < 买点 | 09:30 不动；09:45 已过仍未触发 → **放弃，不追** |
| 💥 持仓异动 | 持仓股盘中 -3% 或放量 2x 砸盘 | 警报：检查止损线 |

消息面影响叠加：命中持仓的利好/利空消息单列。

### Step 3B · 叙事分支（11:30 / 14:30）

**3B.1 上午/全日定调（一段叙事，4-6 句）**

要素：
- 涨停数 vs 昨日（增减说明情绪方向）
- 最高连板梯队完整性（断板 = 顶部信号）
- 炸板数（高炸板 = 接力意愿弱）
- 主线题材是否兑现（昨日 L1 推的题材今日表现）
- 隔夜外围 / 上午突发消息的实际影响

每个数字必须带"so what"解读，禁止纯罗列。

**3B.2 题材延续性判定**（沿用 L4 3a）

对今日 reason tag / 概念板块 Top 5，每个判定 延续 / 退潮 / 新主线。

**3B.3 持仓 + 观察池现状**（逐只一行）

呼应 Step 3A 的状态机，但更精简——叙事卡片里只列关键状态变化，不堆全表。

**3B.4 决策建议**

- 11:30 卡片：**下午策略三选一**（保持 / 减仓只盯低吸 / 直接空仓等明日）
- 14:30 卡片：**收盘动作三选一 + 明日预演**（再战 / 减仓观望 / 空仓）+ 3 条明早 L1 重点盯点

## Step 4 · 输出 Telegram 卡片

# 强约束

1. **时点路由是硬约束**：不在 4 个时点窗口内调用 → 拒绝执行，不要"凑合给一份"。
2. **观察池 + 持仓必须双轨**：观察池来自 push_log 解析（L1 推过的），持仓来自 holdings.yaml（用户手动维护）。两者并行盯，不混淆。
3. **消息面是每个时点必做项**（继承 [[feedback-news-awareness]]），不是可选——题材轮动派必须看消息。
4. **纪律卡片要快、要密集**：09:30/09:45 卡片字数 ≤ 1200 字，状态机式输出，不写散文。
5. **叙事卡片要 so what**（继承 [[feedback-postmarket-style]]）：列数字必须解读；持仓处理穷举状态；下午/明日决策三选一不允许模糊措辞。
6. **假突破已破止损但用户没出**：必须在卡片里明写"应立即止损"，不要委婉。
7. **新闻必须带 URL + 时间**；24h 窗口；禁用淘股吧/股吧/微博/雪球热议。
8. **首板候选派盘中提醒**：派别 D 票如果上午同概念已现 2+ 首板，11:30 卡片要明确标"可关注盘中封板时机"。
9. 卡片长度：纪律卡 ≤ 1200 字，叙事卡 ≤ 2200 字。
10. **不在 4 时点外触发**：用户问"现在情况"如果当前不在窗口内，路由直接拒绝并解释。
